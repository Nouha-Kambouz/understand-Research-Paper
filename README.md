# 📄 ResearchPal — Adaptive Research Paper AI

ResearchPal is an AI-powered web application designed to simplify and explain research papers using modern NLP and prompt engineering techniques.

The system allows users to:

* Ask questions about research papers
* Generate adaptive summaries
* Compare prompting strategies
* Visualize concepts through interactive mind maps

ResearchPal adapts explanations according to the user's expertise level:

* 🌱 Beginner
* 📖 Intermediate
* 🔬 Advanced

The project demonstrates the practical use of:

* Large Language Models (LLMs)
* Prompt Engineering
* Zero-shot Prompting
* Chain-of-Thought Prompting
* Few-shot Prompting
* NLP-based Summarization
* Knowledge Extraction
* Readability Evaluation

---

# ✨ Features

## 📄 Research Paper Analysis

Paste a research paper abstract or full text and interact with it using AI.

## 🎓 Adaptive Explanation Levels

The system adjusts vocabulary and explanation complexity depending on the selected user level.

## 🧠 Prompt Engineering Strategies

ResearchPal supports:

* Zero-shot prompting
* Chain-of-Thought prompting
* Few-shot prompting

## 📋 Adaptive Summarization

Generate structured summaries including:

* TL;DR
* Problem
* Method
* Results
* Real-world impact

## ⚖️ Prompt Strategy Comparison

Compare how different prompting strategies generate different answers for the same question.

## 🗺️ Interactive Mind Map

Automatically extracts important concepts from the paper and visualizes them in an interactive mind map.

## 📊 Readability Metrics

The application evaluates generated text using the Flesch-Kincaid readability score.

# 🏗️ System Architecture

```text
User Interface (HTML/CSS/JavaScript)
                ↓
Prompt Engineering Layer
                ↓
OpenRouter API (Claude Sonnet)
                ↓
Response Processing Layer
                ↓
Evaluation Metrics & Visualization
                ↓
User Interface Rendering
```

---

# 🧠 How the System Works

1. The user pastes a research paper.
2. The user selects:

   * expertise level
   * prompting strategy
3. The system dynamically builds a prompt.
4. The prompt is sent to the Claude model through OpenRouter API.
5. The AI generates adaptive responses.
6. The system evaluates and displays the output.
